# Wireshark Network Traffic Analysis

## Objective

Analyze network traffic to identify suspicious activity using Wireshark.

## Tool Used

Wireshark (Windows)

## Method

1. Started packet capture in Wireshark.
2. Generated normal browsing traffic.
3. Applied filters to analyze traffic.

## Filters Used

dns
http
tcp

## Observations

* DNS queries were made to resolve domain names.
* HTTP traffic showed communication between client and web server.
* No suspicious traffic detected during analysis.

## Skills Demonstrated

* Packet capture
* Network traffic analysis
* Basic threat detection

## Conclusion

Wireshark helps security analysts monitor and analyze network traffic to detect potential threats or suspicious activity.
